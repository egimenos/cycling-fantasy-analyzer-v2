## Work Package WP02: BPI Domain Service — Signal Computation (Priority: P0)

**Goal**: Implement the 5 individual BPI signals as pure functions in a new `domain/breakout/` module, plus the composite index calculation.
**Independent Test**: Each signal function returns expected scores for known inputs. `computeBpiIndex()` sums signals correctly and clamps to 0-100.
**Prompt**: `/tasks/WP02-bpi-signal-computation.md`
**Estimated Size**: ~480 lines

### Included Subtasks

- [ ] T004 Create `apps/api/src/domain/breakout/breakout.types.ts` with `ComputeBreakoutInput` interface
- [ ] T005 Implement `computeTrajectory(seasons, age): number` — linear regression slope × age factor (0-25)
- [ ] T006 Implement `computeRecencyBurst(seasons): number` — current season vs historical avg (0-25)
- [ ] T007 Implement `computeCeilingGap(seasons, prediction, age): number` — peak vs prediction, age-gated (0-20)
- [ ] T008 Implement `computeRouteFit(categoryScores, profileSummary): number` — dot product (0-15)
- [ ] T009 Implement `computeVariance(seasons): number` — coefficient of variation (0-15)
- [ ] T010 Implement `computeBpiIndex(signals): number` — sum and clamp 0-100

### Implementation Notes

- All functions go in `apps/api/src/domain/breakout/breakout.service.ts`.
- Each signal is a standalone pure function — no class instantiation, no DI.
- Age calculation: `(referenceDate - birthDate) / (365.25 * 86400000)`. Default to 28 if birthDate is null.
- Trajectory uses simple linear regression (slope of season totals over years), NOT weighted by temporal decay.
- Recency requires current season total > 20 pts to activate (avoid noise from riders with 3 early-season points).
- Ceiling gap is disabled (returns 0) when age > 33.
- Route fit returns 0 when profileSummary is not provided.
- Variance defaults to 7.5 (midpoint) when fewer than 2 non-zero seasons.

### Parallel Opportunities

- T005 through T009 are independent signal implementations — all can proceed in parallel.
- T010 depends on T005-T009 (needs the signal type).
- T004 should be done first (defines the input shape).

### Dependencies

- Depends on WP01 (needs shared types and RiderProps.birthDate).

### Risks & Mitigations

- Linear regression with 1 point → slope is 0 (intentional: trajectory signal defaults to 0 for single-season riders).
- Division by zero in recency/variance → guard with `if (denominator === 0) return 0`.

**Requirements Refs**: FR-001, FR-007

## Work Package WP03: BPI Domain Service — P80, Flags & Integration (Priority: P0)

**Goal**: Complete the BPI computation with upside P80, flag evaluation, and the composed `computeBreakout()` function. Wire it into the analyze use case. Deliver 100% test coverage.
**Independent Test**: Submit a price list via POST `/api/analyze` and verify each matched rider has a `breakout` field with `index`, `upsideP80`, `flags`, and `signals`. Run `jest --coverage --testPathPattern=breakout` and verify 100% line coverage.
**Prompt**: `/tasks/WP03-bpi-p80-flags-integration.md`
**Estimated Size**: ~450 lines

### Included Subtasks

- [ ] T011 Implement `computeUpsideP80(seasons, prediction): number` — bootstrap (≥3 seasons) or heuristic (<3)
- [ ] T012 Implement `evaluateFlags(input): BreakoutFlag[]` — 6 flag conditions per spec
- [ ] T013 Compose `computeBreakout(input): BreakoutResult` orchestrating signals + P80 + flags
- [ ] T014 Unit tests for all functions in `apps/api/src/domain/breakout/__tests__/breakout.service.spec.ts` — 100% coverage
- [ ] T015 Integrate `computeBreakout()` call into `apps/api/src/application/analyze/analyze-price-list.use-case.ts` (step 5.5, post-ML enrichment)
- [ ] T016 Compute `medianPtsPerHillio` across the rider list for DEEP_VALUE flag evaluation

### Implementation Notes

- **P80 bootstrap**: Sample 1000 iterations from season totals with replacement, apply temporal weights, take the 80th percentile. For <3 seasons, use `prediction × 1.8`.
- **Flag conditions** (from spec):
  - EMERGING_TALENT: age < 25 AND seasonsUsed <= 3 AND raw trajectory slope > 30 pts/year
  - HOT_STREAK: current season total > 2× average of other seasons
  - DEEP_VALUE: price <= 100 AND pointsPerHillio > median
  - CEILING_PLAY: peak historical season > 5× current prediction AND age < 30
  - SPRINT_OPPORTUNITY: price <= 125 AND sprint+stage % > 15% AND flat stage % > 35% (flat = p1Count/totalStages from profileSummary)
  - BREAKAWAY_HUNTER: price <= 100 AND mountain % > 10%
- **Use case integration**: After ML enrichment loop (around line 250), compute `medianPtsPerHillio` once, then `map()` over matched riders calling `computeBreakout()` for each. Assign result to `rider.breakout`. Unmatched riders get `breakout: null`.
- **Tests**: Cover each signal, P80 both paths, each flag independently, edge cases (1 season, 0 points, null birthDate, no profileSummary).

### Parallel Opportunities

- T011, T012 are independent of each other.
- T013 depends on T011+T012.
- T014 can start once T013 is complete.
- T015+T016 are the integration step — depend on T013.

### Dependencies

- Depends on WP02 (signal functions must exist).

### Risks & Mitigations

- Bootstrap randomness → seed the RNG in tests for deterministic assertions.
- Use case modification risk → the integration is additive (new step after existing flow), no existing logic changes.

**Requirements Refs**: FR-001, FR-002, FR-003, FR-005, FR-006, FR-007
