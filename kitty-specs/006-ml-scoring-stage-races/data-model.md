# Data Model: ML Scoring for Stage Races

**Feature**: 006-ml-scoring-stage-races
**Date**: 2026-03-20

## New Entity: ML Score

### Table: `ml_scores`

| Column          | Type      | Constraints                   | Description                                                  |
| --------------- | --------- | ----------------------------- | ------------------------------------------------------------ |
| id              | UUID      | PK, default gen_random_uuid() | Primary key                                                  |
| rider_id        | UUID      | FK → riders.id, NOT NULL      | The rider this prediction is for                             |
| race_slug       | VARCHAR   | NOT NULL                      | Race identifier (e.g., "tour-de-france")                     |
| year            | INTEGER   | NOT NULL                      | Race year                                                    |
| predicted_score | REAL      | NOT NULL                      | ML model predicted score (total projected points)            |
| model_version   | VARCHAR   | NOT NULL                      | Timestamp-based version identifier (e.g., "20260320T030000") |
| created_at      | TIMESTAMP | NOT NULL, default now()       | When this prediction was generated                           |

### Constraints

- **UNIQUE**: `(rider_id, race_slug, year, model_version)` — one prediction per rider per race per model version
- **INDEX**: `(race_slug, year, model_version)` — fast lookup for "all predictions for a race with latest model"
- **INDEX**: `(model_version)` — fast filter by model version for cleanup/comparison

### Relationships

```
riders (existing)
  │
  └─── ml_scores (NEW)
         rider_id FK → riders.id
```

No relationship to `race_results` or `startlist_entries` — the Python pipeline joins those tables during feature extraction, but `ml_scores` only stores the final prediction output.

## Modified Entity: AnalyzedRider (shared-types)

### New Fields

| Field              | Type                  | Description                                                 |
| ------------------ | --------------------- | ----------------------------------------------------------- |
| scoring_method     | `"rules" \| "hybrid"` | Which scoring method produced this result                   |
| ml_predicted_score | `number \| null`      | ML model prediction (null for classics or when unavailable) |

### Unchanged Fields

All existing fields remain: `compositeScore`, `totalProjectedPts`, `categoryScores` (gc, stage, mountain, sprint), `pointsPerHillio`, `seasonsUsed`, `seasonBreakdown`, etc.

## Modified Entity: BenchmarkResult (domain)

### New Fields

| Field             | Type             | Description                                                               |
| ----------------- | ---------------- | ------------------------------------------------------------------------- |
| mlSpearmanRho     | `number \| null` | Spearman rho using ML predictions (null for classics)                     |
| hybridSpearmanRho | `number \| null` | Spearman rho using hybrid method (ML for stage races, rules for classics) |

### Unchanged Fields

Existing `spearmanRho` is renamed to `rulesSpearmanRho` for clarity. All other fields remain.

## Model Artifact (filesystem, not DB)

### Path: `ml/models/`

| File                      | Format           | Description                      |
| ------------------------- | ---------------- | -------------------------------- |
| `model_mini_tour.joblib`  | joblib (sklearn) | Trained RF model for mini tours  |
| `model_grand_tour.joblib` | joblib (sklearn) | Trained RF model for grand tours |
| `model_version.txt`       | plain text       | Current model version identifier |

Models are gitignored. Generated by `make retrain`, consumed by `make predict`.

## Data Flow

```
                 Python Pipeline                    PostgreSQL                  TypeScript API
                 ──────────────                    ──────────                  ──────────────
make retrain  →  1. Load race_results + riders
                    + startlist_entries
                 2. Extract 36 features (pandas)
                 3. Train RF per type
                 4. Save model.joblib         →     ml/models/
                 5. For each race with startlist:
                    Extract features for riders
                    model.predict(features)    →    INSERT ml_scores
                                                                         →   SELECT ml_scores
                                                                              WHERE race_slug, year
                                                                              ORDER BY model_version DESC
                                                                              LIMIT 1 per rider
                                                                         →   Enrich AnalyzedRider
                                                                              with ml_predicted_score
```
