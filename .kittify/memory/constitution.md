## TypeScript Conventions

### Tooling Setup

All packages must include:
- **TypeScript** in strict mode (`"strict": true`)
- **ESLint** with TypeScript plugin
- **Prettier** for formatting
- **Husky** + **lint-staged** for pre-commit hooks

### General Best Practices

- **Immutability by default**: Use `const`, `readonly`, and `as const` whenever possible. Avoid direct mutation; prefer spread, `.map()`, `.filter()`
- **Avoid negation in boolean names**: Use `isActive` not `isNotDisabled`
- **Discriminated unions to model state**:
  ```typescript
  type AsyncState<T> =
    | { status: "idle" }
    | { status: "loading" }
    | { status: "success"; data: T }
    | { status: "error"; error: string };
  ```

### Naming Conventions

| Construct | Convention | Example |
|-----------|-----------|---------|
| Variables & functions | `camelCase` | `getUserProfile`, `isActive` |
| Types, interfaces, classes | `PascalCase` | `UserProfile`, `OrderService` |
| Global/immutable constants | `UPPER_SNAKE_CASE` | `MAX_RETRY_ATTEMPTS` |
| Enum names & members | `PascalCase` | `OrderStatus.Confirmed` |
| Files & folders | `kebab-case` | `user-profile.ts`, `order-service.ts` |

- **Do not prefix interfaces with `I`** — use `UserProfile` not `IUserProfile`

### Type System

- **Prefer `interface` for objects** (extensible, better compiler performance); **`type` for unions and compositions**
- **Avoid `any`** — use `unknown` with type guards instead
- **Always use `strict` mode** — no implicit `any`, no loose null checks
- **Explicit return types** on all exported/public functions
- **Leverage utility types**: `Partial<T>`, `Required<T>`, `Pick<T,K>`, `Omit<T,K>`, `Readonly<T>`, `Record<K,V>`, `ReturnType<F>`, `Parameters<F>`

```typescript
// Correct
interface User {
  id: string;
  email: string;
}
type Result<T> = { success: true; data: T } | { success: false; error: string };

export async function fetchUsers(): Promise<User[]> { ... }

// Incorrect
interface IUser { ... }           // No "I" prefix
function parse(input: any): any   // No any
```
